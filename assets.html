Pinlist Asset Map

This page shows how the project pieces connect from UI to ML-ready backend data.

Frontend Shell

index.html bootstraps the SPA with login/board state sections and links to css/style.css for layout.

It loads js/main.js to decide whether to show the login call-to-action or render Pinterest boards.

Auth Helpers

js/auth.js keeps Pinterest OAuth logic: redirecting users, validating the callback state, exchanging codes via the backend, and storing the session ID.

Once authenticated, the session ID drives every subsequent call to the Flask API.

Flask Backend

python_server/server.py maintains an in-memory token store keyed by session ID, proxies Pinterest API calls, paginates boards/pins, and exposes structured endpoints.

/boards returns board metadata, /pins/board_id returns curated pin info, and /board-data/board_id bundles everything needed for the CLIP/Text pipeline.

Supportive Server

server/index.js is a lightweight Express alternative that only exposes /exchange-token for Pinterest token swaps if you prefer Node.

It demonstrates the bare minimum headers/body structure needed to talk to Pinterest’s OAuth endpoint.

1. Browser → loads index.html → runs js/auth.js + js/main.js
2. Auth flow → js/auth.js calls Flask /exchange-token or Express /exchange-token
3. Data fetch → js/main.js requests /boards and later /board-data/<board_id>
4. ML pipeline → Flask replies with pins, image URLs, and assembled text used for embeddings/CLIP.
